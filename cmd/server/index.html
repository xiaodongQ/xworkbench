🔀 代理

⚡ 命令执行 ▶ 点击展开 API 说明
接口：POST http://xworkbench:8902/api/exec （端口为服务实际监听端口，即登录页地址中的端口）
功能：在 xworkbench 所在机器上执行指定命令，返回 stdout/stderr/exit_code
应用场景：远程触发脚本执行（如备份、重启服务、部署）、调用本地工具（python/go/二进制工具）
参数说明：
AuthorizationBearer <api_key> 或 X-API-Key 头必填（默认 key：xworkbench）
command要执行的命令字符串必填
cwd工作目录（影响相对路径解析）可选
timeout_ms超时时间（毫秒）可选，默认 30000
请求示例：
curl -X POST http://xworkbench:8902/api/exec \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer xworkbench" \
  -d '{"command": "python /opt/scripts/backup.py --target db1", "cwd": "/opt", "timeout_ms": 60000}'
响应字段：output（stdout）、error_out（stderr）、exit_code、duration_ms
🌐 HTTP 代理转发 ▶ 点击展开 API 说明
接口：POST http://xworkbench:8902/api/relay/proxy （端口为服务实际监听端口，即登录页地址中的端口）
功能：透传 HTTP 请求到目标 URL，自动携带 authorization/x-api-key/x-auth-token/cookie 等认证头
应用场景：跨网络调用第三方 API（如内部服务无外网权限、通过 xworkbench 代理访问）、聚合多个 API 请求
参数说明：
AuthorizationBearer <api_key> 或 X-API-Key 头必填（默认 key：xworkbench）
url目标 URL（必填，绝对路径）必填
methodHTTP 方法 GET/POST/PUT/DELETE可选，默认 GET
headers自定义请求头（仅认证头会被转发）可选
body请求体（POST/PUT 时）可选
timeout_ms超时时间（毫秒）可选，默认 30000
请求示例：
curl -X POST http://xworkbench:8902/api/relay/proxy \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer xworkbench" \
  -d '{"method": "POST", "url": "https://api.internal.com/data", "body": "{\"key\":\"value\"}", "timeout_ms": 30000}'
响应字段：status_code、headers、body（响应体）、error
