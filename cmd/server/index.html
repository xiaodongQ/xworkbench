⚠️ 完全放开 CLI 权限

开启后，AI CLI（claude / cbc）的命令将以 --dangerously-skip-permissions 运行， 撤销所有 --allowedTools 白名单，AI 可执行任意 Bash 命令、读写任意路径、跳过所有权限确认。

  • 评估员（evaluator）不受此开关影响，始终使用严格白名单
  • shell 类型任务不受影响（shell 本身无权限概念）
  • 关闭后立即恢复默认白名单 (Bash,Write,Edit,Read,Grep)

🔀 代理 执行命令 / 转发请求

⚡
命令执行
在本机执行 shell / powershell，返回 stdout / stderr / exit_code
▶ 接口说明（点击展开）
接口：POST /api/exec
功能：在 xworkbench 所在机器执行命令，返回 stdout/stderr/exit_code
场景：远程触发脚本（备份 / 重启 / 部署）、调用本地工具
参数：
  • command 必填，命令字符串
  • cwd 可选，工作目录
  • timeout_ms 可选，默认 30000
  • Authorization 必填，Bearer <api_key>
示例：
curl -X POST /api/exec \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer xworkbench" \
  -d '{"command":"ls -la","cwd":"/","timeout_ms":5000}'
🌐
HTTP 代理转发
透传 HTTP 请求到目标 URL，自动携带认证头
▶ 接口说明（点击展开）
接口：POST /api/relay/proxy
功能：透传 HTTP 请求到目标 URL，自动携带 authorization/x-api-key/x-auth-token/cookie 等认证头
场景：跨网络调第三方 API（内部服务无外网 / 聚合多个 API）
参数：
  • url 必填，目标 URL
  • method 可选，默认 GET
  • headers 可选，自定义请求头（仅认证头会被转发）
  • body 可选，POST/PUT 请求体
  • timeout_ms 可选，默认 30000
  • Authorization 必填，Bearer <api_key>
示例：
curl -X POST /api/relay/proxy \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer xworkbench" \
  -d '{"method":"POST","url":"https://api.example.com/data","body":"{}","timeout_ms":30000}'

功能说明
AI 自治让 AI 自动从任务池认领任务并执行，形成自动 → 评估 → 学习的闭环。 启用后，任务详情弹窗中会出现 AI 自治区块，支持三个动作：
▶ run-loop — 让 AI 自动循环执行任务，直到达到阈值分数
📊 reevaluate — 重新评估任务最新执行结果
用法说明
开启后，打开任意任务的详情弹窗，可见 AI 自治区块。执行结果会自动评估打分，优质经验自动入库。
💡 提示：评估依赖 AI 模型，返回结果可能需要 5-30 秒
提示：位置文件控制 config.json → 顶层 ai_loop_enabled
